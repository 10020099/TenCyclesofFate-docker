# 🌟 God Mode 使用指南

> 本指南面向完全没有编程经验的玩家，手把手教你使用 God Mode 的特殊功能。

---

## 一、如何开启 God Mode？

1. 找到项目中的 `backend/.env` 文件（如果没有，复制 `backend/.env.example` 并重命名）
2. 找到这一行：
   ```
   PLAYER_ID="player"
   ```
3. 把它改成：
   ```
   PLAYER_ID="god"
   ```
4. 保存文件，重启游戏服务器

现在你就是"神"了！

---

## 二、God Mode 有什么特权？

### ✨ 特权 1：所有判定必定大成功

游戏中的骰子判定（比如战斗、炼丹、破阵等）会 100% 大成功，再也不用担心运气不好了。

### ✨ 特权 2：可以直接修改游戏数据

在游戏的输入框里，你可以输入特殊指令来修改任何数值。

---

## 三、指令使用教程

### 📖 查看帮助

在输入框输入：
```
/help
```
会显示所有可用指令的说明。

---

### 📖 查看当前数据

**语法：** `/get 你想看的东西`

#### 例子 1：查看角色全部信息
```
/get current_life
```

#### 例子 2：只看属性
```
/get current_life.属性
```

#### 例子 3：看剩余机缘次数
```
/get opportunities_remaining
```

---

### 📖 修改数据

**语法：** `/set 要改的东西 新的值`

#### 例子 1：把灵石改成 99999
```
/set current_life.灵石 99999
```

#### 例子 2：把生命值改成 1000
```
/set current_life.生命值 1000
```

#### 例子 3：把根骨属性改成 100
```
/set current_life.属性.根骨 100
```

#### 例子 4：把剩余机缘改成 99 次
```
/set opportunities_remaining 99
```

#### 例子 5：直接结束当日试炼（标记为已完成）
```
/set daily_success_achieved true
```

---

### 📖 增加或减少数值

**语法：** `/set 要改的东西 +数字` 或 `/set 要改的东西 -数字`

#### 例子 1：灵石增加 5000
```
/set current_life.灵石 +5000
```

#### 例子 2：生命值减少 50
```
/set current_life.生命值 -50
```

#### 例子 3：机缘次数增加 5
```
/set opportunities_remaining +5
```

---

### 📖 重置游戏

如果想从头开始今天的游戏：
```
/reset
```
执行后刷新页面即可。

## 四、常用路径速查表

| 你想改什么 | 路径怎么写 |
|-----------|-----------|
| 灵石数量 | `current_life.灵石` |
| 生命值 | `current_life.生命值` |
| 最大生命值 | `current_life.最大生命值` |
| 根骨属性 | `current_life.属性.根骨` |
| 悟性属性 | `current_life.属性.悟性` |
| 气运属性 | `current_life.属性.气运` |
| 剩余机缘次数 | `opportunities_remaining` |
| 是否在试炼中 | `is_in_trial` |
| 今日是否已完成 | `daily_success_achieved` |

## 五、注意事项

1. **必须先开始试炼**：`current_life` 相关的数据只有在试炼开始后才存在，否则会提示路径不存在。

2. **路径要写对**：中间用英文的点 `.` 连接，不要用中文的句号 `。`

3. **数字不要加引号**：
   - ✅ 正确：`/set current_life.灵石 99999`
   - ❌ 错误：`/set current_life.灵石 "99999"`

4. **指令区分大小写**：`/set` 和 `/SET` 都可以，但路径中的中文要完全一致。

5. **不确定路径？先用 /get 看看**：
   ```
   /get current_life
   ```
   这会显示所有可用的字段名称。

---

## 六、遇到问题？

### 问题：提示"路径不存在"
- 原因：你还没开始试炼，或者路径写错了
- 解决：先开始一次试炼，然后用 `/get current_life` 查看正确的字段名

### 问题：修改后数值没变化
- 原因：可能是路径写错了
- 解决：用 `/get` 确认路径是否正确

### 问题：指令没有反应
- 原因：你的 PLAYER_ID 可能不是 "god"
- 解决：检查 `backend/.env` 文件中的 `PLAYER_ID` 设置
